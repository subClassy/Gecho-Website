An Assistant made Controllable Via Hand Gestures

Simple. Easy. Smart.

"Gecho" is a Gesture Assistant which provides the same functionality like that of a mainstream Voice Assistant including Smart Home features. Only difference is that even Mute People can also avail the wide variety of features these Assistants can offer.

Custom Gestures

"Gecho" supports its own custom gestures which are catalogued and discussed about in the further sections.

Functionality

Currently "Gecho" supports 5 gestures that are being identified using Machine Learning Algorithms to perform basic Smart Home features like Controls of Light Switches etc.

Standalone Product

"Gecho" is a complete product in it itself, using Raspberry-Pi and its modules like Temperature Sensor, Sound Sensor, Webcam, etc.

List of Compatible Gestures

The following list contains the description of the gestures "Gecho" is currently capable to identify.

Single Clap

A single clap is used to invoke "Gecho" and allow camera module to take a snap. It works much like "Ok Google" for a Google Assistant. The intesity of the clap is measured by a Condensor Mic.

Open Palm

An "Open Palm" gesture indicates "Gecho" to turn the lights on. For this you need to present an open palm to the camera. It then verfies the gesture and then turns on the Light.

Closed Fist

To switch off the lights a "Closed Fist" gesture is used to indicate the R-Pi to turn the lights off. A closed fist facing towards the camera will do the job here.

Time

A finger pointing towards the wrist of the other hand result in the "Gecho" to identify the gesture to ask time. "Gecho" then calculates the current time and speaks it through the speakers.

Music Play/Pause

Two open palms at an angle of 180 degrees with each other pointing towards the sides is used to play or pause the music. This is implemented using a pygame library which checks whether music is currently played and then reverses the state.

Temperature

A hotline bling celebration action invokes the Temperature module to record the room's temperature and humidity and then speak it using the speakers.

Process of Image Identification

To train our dataset and divide them into buckets as image classifiers we have used a Convolutionsl Neural Network (CNN). We have used a MobileNet pretrained model so as to ensure quicker training but at the cost of some accuracy.


So basically after we have trained the the dataset the new image that is snapped is taken and matched against all these existing trained gestures and the one with the maximum match percentage is chosen and send as input to the "Gecho"
